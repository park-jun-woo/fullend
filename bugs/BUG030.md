# BUG030 — POST JSON body 필드를 c.Query()로 읽음

## 증상
POST 엔드포인트에 JSON body로 전송된 필드가 `c.Query()`로 생성되어 빈 문자열이 됨.

## 재현
SSaC에서 `@call`만 있고 `@post`(DDL model)가 없는 함수에서 `request.cron`을 참조하면, codegen이 `c.Query("cron")`을 생성.

## 기대 동작
OpenAPI에 `requestBody` + `application/json`이 정의된 POST 엔드포인트는 `ShouldBindJSON`으로 JSON body를 읽어야 함.

## 원인 추정
`@post` DDL model이 있을 때만 JSON body 바인딩이 생성되는 로직. `@call`만 있는 경우에도 OpenAPI requestBody가 있으면 JSON body 바인딩이 필요.
