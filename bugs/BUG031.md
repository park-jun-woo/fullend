# BUG031 — fullend.yaml session.backend 설정 시 main.go에 session.Init() 미생성

## 증상
`fullend.yaml`에 `session.backend: postgres`가 있지만, 생성된 `main.go`에 `session.Init()` 호출이 없음. `pkg/session`의 `defaultModel`이 nil 상태.

## 기대 동작
`queue.Init()`처럼 main.go에서 `session.Init(session.NewPostgresSession(ctx, db))` 또는 `session.Init(session.NewMemorySession())`을 자동 생성해야 함.

## 원인
Phase051에서 `pkg/session`에 `Init()` + `@func` 래퍼를 추가했지만, gluegen의 main.go 생성 로직에 session/cache/file Init 주입이 아직 구현되지 않음.
