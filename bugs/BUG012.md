# BUG012: ssac-gen 미사용 import 생성 — 빌드 실패

## 단계
ssac-gen

## 심각도
HIGH — 컴파일 에러

## 증상
`go build` 시 `imported and not used` 에러로 빌드 실패.

## 에러 목록

```
internal/service/auth/register.go:4:2: "database/sql" imported and not used
internal/service/project/create_project.go:5:2: "database/sql" imported and not used
internal/service/project/delete_project.go:5:2: "database/sql" imported and not used
internal/service/project/list_projects.go:7:2: "strconv" imported and not used
internal/service/project/update_project.go:5:2: "database/sql" imported and not used
internal/service/deploy/deploy_project.go:5:2: "database/sql" imported and not used
internal/service/schedule/create_schedule.go:5:2: "database/sql" imported and not used
internal/service/schedule/delete_schedule.go:5:2: "database/sql" imported and not used
internal/service/schedule/update_schedule.go:5:2: "database/sql" imported and not used
internal/service/chat/send_message.go:5:2: "database/sql" imported and not used
```

## 영향 범위
- `"database/sql"` — 10개 서비스 파일 (handler.go 제외한 거의 모든 서비스)
- `"strconv"` — list_projects.go

## 원인
ssac-gen이 서비스 파일 생성 시 `database/sql`과 `strconv`을 일괄 import하지만, BUG005(중복 파싱 제거), BUG009(rows.Err), BUG010(@empty 가드) 등 수정으로 해당 패키지를 실제 사용하는 코드가 제거/변경되면서 import만 남음.

## 수정 제안
- ssac-gen에서 import를 사용 여부에 따라 조건부 생성, 또는
- 생성 후 `goimports` 실행으로 미사용 import 자동 정리
