# BUG020: @empty on int64 필드 — nil 비교 코드 생성

## 단계
gen → go build

## 재현
- SSaC: `// @empty credits.Balance "Insufficient credits" 402`
- `credits.Balance`가 `int64` 타입일 때 생성 코드:
  ```go
  if credits.Balance == nil {  // compile error: int64 vs nil
  ```

## 원인
`@empty` 코드 생성기가 필드 타입을 확인하지 않고 무조건 `== nil` 비교를 생성한다. 포인터/인터페이스가 아닌 값 타입(int64, string 등)에 대해서는 zero value 비교(`== 0`, `== ""`)를 생성해야 한다.

## 기대 동작
- 포인터/인터페이스 → `== nil`
- int64 → `== 0`
- string → `== ""`

## 우회
`@empty` 대신 `@call` func에서 검증 후 에러 반환 (`@error 402`)
