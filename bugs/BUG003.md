# BUG003: ssac-gen @post 인자를 알파벳순으로 배치 — SQL 파라미터 순서와 불일치

## 단계
ssac-gen

## 증상
런타임 500 에러. 인자 타입이 같으면 컴파일은 통과하지만 값이 뒤바뀜.

## 재현
`users` DDL: `(email, password_hash, name)` 순서.
SSaC `register.ssac`에서 `@post User user = User.Create({Email: request.email, PasswordHash: hp.HashedPassword, Name: request.name})` 실행 시.

## 생성된 코드

```go
// internal/service/auth/register.go
user, err := h.UserModel.Create(email, name, hp.HashedPassword)
//                               ↑ $1   ↑ $2 → name이 password_hash 위치에 들어감
```

## 모델 시그니처

```go
// internal/model/user.go — SQL 파라미터 순서 기준
func (m *userModelImpl) Create(email string, passwordHash string, name string) (*User, error)
//                              $1            $2                   $3
```

## 기대 동작

```go
user, err := h.UserModel.Create(email, hp.HashedPassword, name)
```

## 원인
ssac-gen이 `@post` 시퀀스의 인자를 **알파벳순**으로 정렬하여 호출 코드를 생성하지만, ssac-model은 **SQL 파라미터 순서**($1, $2, $3)대로 시그니처를 생성. 두 gen 단계의 인자 순서 규칙이 다름.

## 영향 범위
타입이 모두 `string`인 경우 컴파일 에러 없이 데이터 오염 발생. `int64`와 `string`이 섞이면 컴파일 에러.

## 수정 제안
ssac-gen에서 모델 메서드를 호출할 때, 알파벳순이 아닌 **모델 시그니처 파라미터 순서**(= SQL 파라미터 순서)에 맞춰 인자를 배치.
