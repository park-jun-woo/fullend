# BUG027: @subscribe message struct가 생성 코드에 미출력

## 발견 경위
zenflow-try06에서 `@subscribe "workflow.executed"` 사용 시, .ssac 파일에 선언한 `WorkflowExecutedMessage` struct가 생성된 Go 코드에 포함되지 않아 빌드 실패.

## 재현
1. .ssac 파일에 message struct 선언 + `@subscribe` + `func OnXxx(message StructName) {}`
2. `fullend gen` 실행
3. 생성된 .go 파일에 struct 정의 없이 `message WorkflowExecutedMessage` 참조만 존재
4. `go build` → `undefined: WorkflowExecutedMessage`

## 기대 동작
매뉴얼: "Message struct must be declared as a Go struct in the same .ssac file"
→ codegen이 해당 struct를 생성 .go 파일에도 출력해야 함

## 우회
생성된 패키지에 수동으로 struct 정의 파일 추가
