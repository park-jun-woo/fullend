# BUG028: FK 참조 @get 후 @empty 강제 규칙이 @exists 패턴과 충돌

## 발견 경위
zenflow-try06에서 `@get Template existing = Template.FindBySourceWorkflowID(...)` 후
`@exists existing "Already published" 409`을 사용하려 했으나,
validator가 FK 참조 조회 후 @empty 가드를 강제 요구.

## 문제
- @empty: nil이면 실패 (404) — "찾지 못했다"
- @exists: not nil이면 실패 (409) — "이미 있다"
- 논리적으로 모순: 동시에 사용 불가

## 기대 동작
@get 후 @exists가 따라오면 @empty 가드 강제 규칙을 면제해야 함.
@exists의 의도는 "이미 존재하면 에러"이므로 nil이 정상 경로.

## 우회
@exists 제거, DB UNIQUE 제약에 의존
