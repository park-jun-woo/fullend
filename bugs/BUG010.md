# BUG010: UpdateSchedule/DeleteSchedule에서 project nil 체크 누락 — 패닉 가능

## 단계
ssac-gen

## 심각도
MEDIUM

## 증상
고아 schedule(프로젝트 삭제 후 남은)에 대해 Update/Delete 시도 시 nil pointer dereference 패닉 → 500.

## 생성된 코드

```go
// internal/service/schedule/update_schedule.go
project, err := h.ProjectModel.FindByID(schedule.ProjectID)
if err != nil { ... }
// project가 nil일 수 있음 (프로젝트 삭제된 경우)
if err := authz.Check(authz.CheckRequest{
    ResourceID: project.ID,  // ← nil 패닉
}); err != nil { ... }
```

## 기대 동작
`project`에 대한 `@empty` 체크가 있어야 함. 또는 schedule 조회 후 project 존재 여부를 확인.

## 영향 범위
update_schedule.go, delete_schedule.go

## 수정 제안
ssac-gen에서 FK 참조로 조회한 엔티티에 대해 자동으로 nil 체크를 생성하거나, SSaC에서 명시적 `@empty` 체크가 없는 경우 경고.
