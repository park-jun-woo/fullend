# BUG013: scenario-gen 캡처 변수명 불일치 + 중복 Register

## 단계
scenario-gen

## 심각도
HIGH — 테스트 실행 불가

## 증상
`hurl --test scenario-project.hurl` 실행 시 `Undefined variable: id` 에러.

## 문제 1: 캡처 변수명 ↔ 사용 변수명 불일치

```hurl
# CreateProject — 캡처
[Captures]
project_project_id: jsonpath "$.project.id"

# GetProject — 사용
GET {{host}}/projects/{{id}}    ← {{project_project_id}} 여야 함
```

캡처는 `project_project_id`로 하지만 URL에서는 `{{id}}`로 참조. `SendMessage`도 동일:
```hurl
POST {{host}}/projects/{{id}}/messages    ← {{project_project_id}} 여야 함
```

## 문제 2: 같은 이메일로 중복 Register

파일 내 두 시나리오("Create and get project", "Send message")가 동일한 `pm@flowave.cloud`로 Register. 두 번째 Register는 unique 제약 위반으로 실패.

## 문제 3: 캡처 변수 네이밍

`project_project_id`는 `{resource}_{field}` 규칙으로 보이는데 `project.id`를 캡처하면 `project_id`가 자연스러움. `project_project_id`는 중복 접두사.

## 수정 제안
1. 캡처 변수를 URL 템플릿에서 정확히 참조하도록 통일
2. 시나리오별 고유 이메일 생성 (예: `pm1@flowave.cloud`, `pm2@flowave.cloud`) 또는 시나리오 간 Register 재사용
3. 캡처 변수 네이밍을 `{resource}_id` 형태로 단순화

## 상태: Won't Fix

Gherkin 파서 + 시나리오 Hurl 변환기 전체 폐기 결정 (Phase009). 시나리오 테스트는 사용자가 `.hurl`을 직접 작성하므로 변환 레이어가 소멸, 버그 원인 자체가 사라짐.
