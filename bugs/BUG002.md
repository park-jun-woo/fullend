# BUG002: ssac-gen ListByXxx pagination 인자 순서 뒤바뀜

## 단계
ssac-gen

## 증상
`go build` 시 `cannot use opts as int64 value in argument` 컴파일 에러.

## 재현
OpenAPI에 `x-pagination`이 있는 List 엔드포인트 + SSaC에서 해당 모델 호출 시 `fullend gen` 실행.

## 생성된 코드

```go
// internal/service/project/list_projects.go
h.ProjectModel.ListByUserID(opts, currentUser.ID)
```

## 모델 인터페이스 시그니처

```go
ListByUserID(userID int64, opts model.QueryOpts) ([]Project, error)
```

## 기대 동작
`h.ProjectModel.ListByUserID(currentUser.ID, opts)` — 비즈니스 파라미터 먼저, QueryOpts 마지막.

## 원인
ssac-gen이 서비스 함수 호출을 생성할 때 pagination QueryOpts를 첫 번째 인자로 배치하지만, ssac-model gen은 비즈니스 파라미터를 먼저 받도록 인터페이스를 생성. 두 gen 단계 간 인자 순서 규칙이 불일치.

## 수정 제안
ssac-gen에서 모델 호출 시 QueryOpts를 항상 마지막 인자로 배치하도록 수정.
