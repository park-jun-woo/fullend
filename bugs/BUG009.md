# BUG009: model list 메서드에서 rows.Err() 미확인

## 단계
ssac-model

## 심각도
MEDIUM

## 증상
DB 이터레이션 중 에러 발생 시 부분 결과를 정상으로 반환. 데이터 누락을 감지하지 못함.

## 생성된 코드

```go
// 예: internal/model/message.go
for rows.Next() {
    v, err := scanMessage(rows)
    if err != nil {
        return nil, err
    }
    items = append(items, *v)
}
return items, nil  // ← rows.Err() 체크 없음
```

## 기대 동작

```go
if err := rows.Err(); err != nil {
    return nil, err
}
return items, nil
```

## 영향 범위
모든 model List/ListBy 메서드: message.go, document.go, deployment.go, schedule.go, project.go

## 수정 제안
ssac-model에서 list 메서드 생성 시 `rows.Close()` defer 이후 `rows.Err()` 체크 코드 추가.
