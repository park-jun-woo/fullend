
## 논문 주제 총정리

### 1. Multi-Artifact Drift Detection

**핵심 명제:** 소프트웨어 시스템은 OpenAPI, DDL, 서비스 로직, 인가 정책, 상태 다이어그램 등 복수의 SSOT가 독립적으로 진화하면서 drift가 발생한다. 기존 도구는 단방향 생성(DDL → OpenAPI)만 지원하며, 이미 살아있는 아티팩트 간 정합성 검증 도구는 존재하지 않는다.

**fullend 기여:** 11종 교차검증 룰로 다중 아티팩트 drift를 실시간 감지. SSaC operationId ↔ OpenAPI, @auth ↔ Rego, @state ↔ Mermaid 등 양방향 검증.

**측정 가능한 것:** drift 발생률, 검출 시점(PR vs 프로덕션), 버그 밀도 상관관계.

### 2. Contract-Based Code Generation with Ownership Directives

**핵심 명제:** 코드 생성 도구의 근본 문제는 "생성 후 수동 수정하면 다음 gen에 덮어써진다"는 딜레마다. 기존 접근은 생성/수동 영역을 파일 단위로 분리하거나(brittle), 전부 재생성하거나(수동 수정 불가)다.

**fullend 기여:** `//fullend:gen`/`//fullend:preserve`를 함수 단위로 박아서 소유권을 코드 안에 내장. contract hash로 SSOT 변경 시 broken 감지. 한 파일에 gen/preserve 혼재 가능.

**측정 가능한 것:** 수동 수정 보존율, 충돌 감지 정확도, 개발자 개입 비용.

### 3. Structural Context Engineering for AI Coding Agents

**핵심 명제:** 기존 컨텍스트 엔지니어링은 프롬프트 레벨에서만 다룬다. "Lost in the Middle", "Context Rot" 연구들이 짧은 컨텍스트의 우위를 증명했지만, 코드 구조 자체를 AI 탐색 동선에 맞게 설계하는 방법론은 존재하지 않는다.

**filefunc 기여:** 1 file 1 concept 원칙으로 read의 단위 = 개념의 단위를 강제. 코드북이 Claude Code의 프로젝트 지도 역할. grep → 어노테이션 메타 판독 → 필요한 파일만 full read의 3단계 파이프라인. 300개 중 290개를 안 여는 구조를 파일시스템 레벨에서 보장.

**측정 가능한 것:** filefunc 적용 전후 Claude Code의 read 횟수, 컨텍스트 토큰 수, 작업 정확도, 탐색 시간 대비 작업 시간 비율.

**강점:** 실험 설계가 깔끔하고 타이밍이 맞다. AI 코딩 도구 전성기에 하드웨어 레이어(파일 구조)를 건드린 첫 논문이 될 수 있다.

### 4. Implicit Coupling Detection via Co-edit History

**핵심 명제:** 정적 분석(calls/uses)은 명시적 의존성만 잡는다. 숨은 의존성(같은 비즈니스 규칙을 다른 각도에서 구현, 암묵적 포맷 계약, 항상 같이 터지는 버그)은 정적 분석으로 검출 불가다. 기존 git history mining 연구는 한 파일에 func이 섞여 있어 노이즈가 크다.

**filefunc/whyso 기여:** func = file 등식이 성립하면 파일 단위 co-edit = 함수 단위 co-edit으로 정확히 떨어진다. 노이즈 없는 함수 단위 coupling 통계가 자동으로 나온다. 명시적 관계 없이 N회 이상 함께 수정되면 자동 WARNING.

**측정 가능한 것:** coupling 검출 정밀도/재현율, 리팩토링 이후 버그 감소율, 기존 git history mining 대비 노이즈 비율.
