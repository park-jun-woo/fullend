# BUG005: list_projects.go에서 sort 쿼리 파라미터 검증 없이 ORDER BY에 삽입 — SQL Injection

## 단계
ssac-gen

## 심각도
HIGH — SQL Injection

## 증상
`GET /projects?sort=id;DROP TABLE users--` 같은 요청으로 임의 SQL 실행 가능.

## 생성된 코드

```go
// internal/service/project/list_projects.go
opts := model.ParseQueryOpts(c, model.QueryOptsConfig{...})
if v := c.Query("sort"); v != "" {
    opts.SortCol = v  // ← 검증 없이 덮어씀
}
if v := c.Query("direction"); v != "" {
    opts.SortDir = v  // ← 검증 없이 덮어씀
}
```

`ParseQueryOpts`에서 allowlist 검증을 하지만, 이후 `c.Query("sort")`로 다시 읽어서 검증 없이 덮어씀.

## 원인
ssac-gen이 `x-pagination`/`x-sort` 처리 시 `ParseQueryOpts`와 별도로 쿼리 파라미터를 재파싱하는 코드를 생성. 이중 파싱에서 allowlist 검증이 빠짐.

## 수정 제안
`ParseQueryOpts` 이후 sort/direction을 재파싱하는 코드를 제거하거나, 재파싱 시에도 allowlist 검증을 적용.
