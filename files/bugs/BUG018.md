# BUG018: v0.1.8 — Terraform 삭제만 했는데 validate에서 새 에러 2건 발생

## 단계
validate (cross-validation)

## 심각도
HIGH — gen 불가 (validate 실패로 코드 생성 차단)

## fullend 버전
v0.1.8

## 증상
v0.1.7에서 0 errors, 0 warnings로 통과하던 동일한 flowave specs가 v0.1.8에서 4 errors, 1 warning으로 실패. v0.1.8의 변경사항은 Terraform 삭제뿐인데 관련 없는 에러가 발생.

## 에러 1: States "required but not found"

```
✗ States       required but not found
```

flowave는 `@state`를 사용하지 않고, states/ 디렉토리도 없음. v0.1.7에서는 문제없이 통과했음. States가 필수로 변경된 이유 불명.

## 에러 2: Hurl 경로 매칭이 path parameter를 인식하지 못함

```
Hurl→OpenAPI: scenario-errors.hurl:48 — path /projects/999999 not found in OpenAPI
Hurl→OpenAPI: scenario-errors.hurl:54 — path /projects/999999/messages not found in OpenAPI
Hurl→OpenAPI: scenario-errors.hurl:64 — path /projects/999999/deploy not found in OpenAPI
Hurl→OpenAPI: scenario-errors.hurl:70 — path /documents/999999 not found in OpenAPI
```

Hurl 테스트에서 `/projects/999999`는 에러 케이스 테스트용 리터럴 경로. OpenAPI에는 `/projects/{id}`로 정의되어 있음. Hurl cross-validator가 path parameter 치환을 하지 않고 리터럴 비교하고 있음.

## 부가 이슈: `--skip scenario`가 Hurl cross-validation을 제외하지 않음

`--skip states,scenario`로 실행해도 Hurl→OpenAPI cross-validation 에러가 여전히 발생하여 gen이 차단됨.

## 기대 동작

1. `@state`를 사용하지 않는 프로젝트에서는 states/ 디렉토리가 없어도 통과
2. Hurl의 `/projects/999999`가 OpenAPI의 `/projects/{id}`에 매칭
3. `--skip scenario` 시 Hurl cross-validation도 함께 스킵

## 재현

```bash
# v0.1.7 → 0 errors, 0 warnings
# v0.1.8 → 아래 결과
fullend validate ~/.clari/repos/flowave/specs
```
