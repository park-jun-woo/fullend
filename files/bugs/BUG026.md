# BUG026: SSaC 파서 — 숫자 리터럴을 변수명으로 인식

## 발견 경위
zenflow-try05에서 `@post Organization.Create({CreditsBalance: 100})` 사용 시
`"100" 변수가 아직 선언되지 않았습니다` 에러 발생.
`@post ExecutionLog.Create({CreditsSpent: 1})` 에서도 동일하게 `"1"` 에러.

## 재현
```go
// @post Organization org = Organization.Create({Name: request.org_name, PlanType: "free", CreditsBalance: 100})
```
→ `"100" 변수가 아직 선언되지 않았습니다`

## 기대 동작
매뉴얼에 숫자 리터럴 지원이 명시됨:
> Numeric: `1`, `42`, `3.14`, `-1`

따라서 `100`도 유효한 숫자 리터럴로 파싱되어야 함.

## 우회 방법
DDL에 `DEFAULT 100`을 설정하고, SSaC @post 인자에서 해당 필드를 제거.

## 영향
숫자 리터럴이 필요한 모든 SSaC 시퀀스에서 발생 가능.
