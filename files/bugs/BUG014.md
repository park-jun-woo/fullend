# BUG014: ssac-gen이 OpenAPI 204 응답을 무시하고 200으로 생성

## 단계
ssac-gen

## 심각도
LOW

## 증상
OpenAPI에 `"204"` (No Content)로 정의된 DELETE 엔드포인트가 `c.JSON(http.StatusOK, gin.H{})` (200)으로 생성됨.

## 재현
OpenAPI:
```yaml
delete:
  operationId: DeleteProject
  responses:
    "204":
      description: Project deleted
```

SSaC:
```go
// @delete Project.Delete({ID: request.id})
// @response! {}
```

생성 코드:
```go
c.JSON(http.StatusOK, gin.H{})  // ← 204여야 함
```

## 기대 동작
OpenAPI 응답 코드가 204일 때 `c.Status(http.StatusNoContent)` 생성.

## 수정 제안
ssac-gen에서 `@response! {}`(빈 응답) + OpenAPI 204 조합일 때 `c.Status(204)` 생성. 또는 `@response` 문법에 상태 코드 지정 옵션 추가 (예: `@response! 204 {}`).
