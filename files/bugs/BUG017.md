# BUG017: ssac-gen이 authz.Check 호출 시 UserID, Role을 전달하지 않음

## 단계
ssac-gen

## 심각도
HIGH — 인가 우회

## fullend 버전
v0.1.6

## 증상
DISABLE_AUTHZ=1 해제 후 authz 활성화하면, 모든 @auth 엔드포인트에서 UserID=0, Role=""로 OPA 평가 → 모든 요청 거부되거나, 소유권 검증이 무의미해짐.

## pkg/authz 구현 확인

`pkg/authz/authz.go`의 `Check()` 함수는 `req.UserID`와 `req.Role`을 OPA input의 claims로 전달:

```go
// pkg/authz/authz.go
type CheckRequest struct {
    Action     string
    Resource   string
    UserID     int64   // ← 필요
    Role       string  // ← 필요
    ResourceID int64
}

func Check(req CheckRequest) (CheckResponse, error) {
    opaInput := map[string]interface{}{
        "claims":      map[string]interface{}{"user_id": req.UserID, "role": req.Role},
        "action":      req.Action,
        "resource":    req.Resource,
        "resource_id": req.ResourceID,
    }
    // ... OPA evaluation with opaInput
}
```

## 생성된 코드 (ssac-gen 출력)

```go
// artifacts/backend/internal/service/project/get_project.go
if _, err = authz.Check(authz.CheckRequest{
    Action:     "read",
    Resource:   "project",
    ResourceID: project.ID,
    // UserID 없음 → 0
    // Role 없음   → ""
}); err != nil {
    c.JSON(http.StatusForbidden, gin.H{"error": "Not authorized"})
    return
}
```

모든 @auth 핸들러에서 동일 패턴 — UserID와 Role이 누락.

## 기대 동작

```go
if _, err = authz.Check(authz.CheckRequest{
    Action:     "read",
    Resource:   "project",
    ResourceID: project.ID,
    UserID:     currentUser.ID,
    Role:       currentUser.Role,
}); err != nil {
    c.JSON(http.StatusForbidden, gin.H{"error": "Not authorized"})
    return
}
```

`currentUser`는 JWT 미들웨어(`middleware.BearerAuth`)가 gin context에 주입하는 `*model.CurrentUser`.

## 영향 범위

@auth가 있는 모든 SSaC 핸들러 (flowave 기준 14개):

- get_project, update_project, delete_project
- list_messages, send_message
- deploy_project, list_deployments
- list_documents
- create_schedule, list_schedules, update_schedule, delete_schedule

## 원인
BUG011에서 "수정 완료"로 표시되었으나, `pkg/authz`의 `CheckRequest` 구조체에 `UserID`, `Role` 필드를 추가한 것까지만 완료. ssac-gen의 `@auth` 코드 생성 템플릿에서 `currentUser.ID`와 `currentUser.Role`을 `CheckRequest`에 포함하는 로직이 누락됨.

## 수정 제안
ssac-gen에서 `@auth` 시퀀스를 `authz.Check` 호출로 변환할 때:
1. `currentUser`를 gin context에서 가져오는 코드가 이미 생성되는지 확인
2. `authz.CheckRequest{}`에 `UserID: currentUser.ID, Role: currentUser.Role` 추가
