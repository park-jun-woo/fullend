# BUG016: hurl-gen이 DELETE 엔드포인트에서 OpenAPI 204 응답코드를 반영하지 않음

## 단계
hurl-gen

## 심각도
MEDIUM — smoke test 실패

## 증상
DELETE /schedules/{id}가 204를 반환하지만, 생성된 smoke.hurl은 `HTTP 200`을 기대하여 테스트 실패.

## 생성된 코드

```hurl
DELETE {{host}}/schedules/{{schedule_id}}
Authorization: Bearer {{token}}

HTTP 200    ← OpenAPI에 204로 정의됨
```

## 기대 동작

```hurl
HTTP 204
```

## 원인
hurl-gen이 모든 응답을 200으로 하드코딩. OpenAPI의 실제 응답 코드를 읽어야 함.

## 수정 제안
hurl-gen에서 OpenAPI 응답 코드(200, 201, 204 등)를 읽어서 smoke.hurl에 반영.
