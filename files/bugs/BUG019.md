# BUG019: sqlFileToModel — 복합 테이블명 inflection 불일치

## 단계
validate

## 재현
- DDL 파일: `execution_logs.sql` (테이블: `execution_logs`)
- SSaC에서 `ExecutionLog.Create(...)` 참조
- `fullend validate` 실행 시 `"ExecutionLog" 모델을 찾을 수 없습니다` 에러 발생

## 원인
`internal/ssac/validator/symbol.go:328` `sqlFileToModel()` 함수가 `strings.ToUpper(singular[:1]) + singular[1:]`로 첫 글자만 대문자화한다.
- `execution_logs` → `inflection.Singular("execution_logs")` → `execution_log` → `Execution_log` (잘못됨)

반면 `internal/gen/gogin/model_impl.go:904` `singularize()`는 `strcase.ToGoPascal()`을 사용하여 올바르게 `ExecutionLog`을 반환한다.

## 기대 동작
`sqlFileToModel()`도 `strcase.ToGoPascal()` 또는 동등한 snake_case → PascalCase 변환을 사용해야 한다.

## 우회
복합 테이블명 대신 단일어 테이블명 사용 (`execution_logs` → `executions`)
