# BUG004: hurl-gen smoke.hurl에서 미정의 변수 {{id}} 사용

## 단계
hurl-gen

## 증상
`hurl --test smoke.hurl` 실행 시 `Undefined variable: id` 에러.

## 재현

```
GET {{host}}/documents/{{id}}    ← {{id}} 캡처된 적 없음
```

## 원인
`GetDocument` 엔드포인트(`/documents/{id}`)가 smoke test에 포함되지만, Document를 생성하는 API가 없어(문서는 에이전트 파이프라인에서 생성) `document_id`를 캡처할 수 없음. hurl-gen이 해당 엔드포인트의 path param `{id}`를 `{{id}}`로 변환했는데, 이전에 캡처된 변수가 없음.

## 수정 제안
- 캡처 의존성 그래프를 분석하여, 필요한 변수를 캡처할 수 없는 엔드포인트는 smoke test에서 제외, 또는
- 독립 path에서 `{id}`를 사용하는 경우 해당 리소스의 Create가 smoke에 없으면 skip 처리
