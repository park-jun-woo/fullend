## 수정 이력

### BUG006 (Phase007)
DDL 컬럼에 `-- @sensitive` 어노테이션 추가 시 `json:"-"` 태그 생성.
crosscheck에서 `password`, `secret`, `hash`, `token` 패턴 컬럼에 `@sensitive` 미선언 시 WARNING.

변경 파일:
- `internal/gluegen/model_impl.go` — `ddlColumn.Sensitive` 필드, `parseDDLFiles()` 파싱, `generateTypesFile()` `json:"-"` 생성
- `internal/crosscheck/sensitive.go` — `ParseSensitive()`, `CheckSensitiveColumns()` 추가
- `internal/crosscheck/crosscheck.go` — `SensitiveCols` 필드 + `CheckSensitiveColumns` 호출
- `internal/orchestrator/validate.go` — `ParseSensitive` 호출 + `SensitiveCols` 전달

### BUG008 (Phase006)
`@error` 어노테이션 파이프라인: `pkg/auth/*.go`의 `// @error 401` → `funcspec.ParseFile` → `gen.go:injectFuncErrStatus()` → SSaC SymbolTable → codegen 시 401 반환.

### BUG009 (Phase007)
`model_impl.go`의 `generateMethodFromIface()` — `isList`와 `isSliceReturn` 분기에서 `for rows.Next()` 루프 직후 `rows.Err()` 체크 추가.

### BUG010 (SSaC 수정지시서006 — 완료)
SSaC에서 validator ERROR 방식 채택: FK 참조 `@get` 후 `@empty` 가드 없으면 ERROR (코드 생성 중단). gigbridge `.ssac` 파일에 `@empty` 가드 추가 필요.
