# BUG006: User 구조체에 password_hash가 json 태그로 노출

## 단계
ssac-model

## 심각도
HIGH — 정보 유출

## 증상
Register, Login, GetMe 응답에 `password_hash` 필드가 포함되어 클라이언트에 bcrypt 해시 노출.

```json
{"user":{"id":1,"email":"test@test.com","password_hash":"$2a$10$...","name":"test",...}}
```

## 생성된 코드

```go
// internal/model/types.go
type User struct {
    ID           int64  `json:"id"`
    Email        string `json:"email"`
    PasswordHash string `json:"password_hash"`  // ← 응답에 포함됨
    Name         string `json:"name"`
    ...
}
```

서비스에서 `c.JSON(200, gin.H{"user": user})` 호출 시 전체 구조체가 직렬화.

## 원인
ssac-model이 DDL 컬럼을 그대로 json 태그에 매핑. `password_hash` 같은 민감 필드를 `json:"-"`로 제외하는 로직 없음.

## 수정 제안
- DDL 컬럼명에 `password`, `secret`, `token`, `hash` 등이 포함되면 `json:"-"` 태그를 자동 적용, 또는
- `@response`에 명시된 필드만 포함하는 DTO를 생성하여 응답에 사용
