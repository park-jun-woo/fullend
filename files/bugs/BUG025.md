# BUG025: @auth 템플릿이 currentUser.ID를 하드코딩

## 원인
@auth 템플릿은 `authz.CheckRequest{UserID: currentUser.ID, Role: currentUser.Role}`로 고정 출력한다.
이는 claims에 `ID`와 `Role` 필드명을 사용하는 것이 규약임을 의미한다.

## 올바른 사용법
```yaml
claims:
  ID: user_id:int64       # 필드명 "ID" 필수 (authz.CheckRequest.UserID 매핑)
  Role: role              # 필드명 "Role" 필수 (authz.CheckRequest.Role 매핑)
```

## 조치
매뉴얼에 `ID`, `Role` 필드명 필수 규약 명시 완료.
