# BUG022: Func → SSaC crosscheck 역방향 매칭 실패 WARNING

## 단계
validate (cross)

## 재현
- SSaC에서 `@call billing.CheckCredits(...)` 참조
- Func spec `func/billing/check_credits.go`에 `// @func checkCredits` 선언
- crosscheck에서 `billing.checkCredits is not referenced by any SSaC @call` WARNING 발생

## 원인
SSaC → Func 순방향 매칭은 정상 작동하나, Func → SSaC 역방향 매칭에서 참조를 찾지 못한다. 대소문자 또는 패키지명 매칭 로직 차이로 추정.

## 영향
WARNING만 발생하므로 validate는 통과하지만, 불필요한 경고가 출력된다.
