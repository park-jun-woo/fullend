# BUG021: BearerAuth 미들웨어 — 커스텀 JWT claims 매핑 누락

## 단계
gen → runtime

## 재현
- `fullend.yaml`에 `OrgID: org_id` claim 설정
- 생성된 `middleware/bearerauth.go`:
  ```go
  c.Set("currentUser", &model.CurrentUser{
      Email: out.Email,
      ID: out.UserID,
      // OrgID: unknown claim key "org_id"  ← 주석 처리됨
      Role: out.Role,
  })
  ```

## 원인
`pkg/auth.IssueToken`과 `VerifyToken`이 `UserID`, `Email`, `Role` 세 필드만 지원한다. `fullend.yaml`에 추가 claims를 정의해도 토큰에 포함되지 않으며, 미들웨어 코드젠에서 주석으로 처리된다.

## 기대 동작
커스텀 claims가 JWT에 포함되고 미들웨어에서 정상적으로 매핑되어야 한다.

## 우회
추가 claims를 사용하지 않고 DB 조회로 대체 (`@get User me = User.FindByID({ID: currentUser.ID})` → `me.OrgID`)
