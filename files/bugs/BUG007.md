# BUG007: authz.Check에 유저 정보(claims) 미전달 — 소유권 검증 불가

## 단계
ssac-gen

## 심각도
HIGH — 인가 우회

## 증상
`@auth` 시퀀스가 있는 모든 핸들러에서 `authz.Check`가 호출되지만, 현재 유저 정보가 전달되지 않아 소유권 검증이 실제로 동작하지 않음.

## 생성된 코드

```go
// 예: internal/service/project/get_project.go
if err := authz.Check(authz.CheckRequest{
    Action:     "read",
    Resource:   "project",
    ResourceID: project.ID,
}); err != nil {
    ...
}
```

OPA 정책에서 `input.claims.user_id`를 비교해야 하는데 claims 자체가 전달되지 않음.

## 기대 동작

```go
authz.Check(authz.CheckRequest{
    Action:     "read",
    Resource:   "project",
    ResourceID: project.ID,
    Claims:     authz.Claims{UserID: currentUser.ID},
})
```

## 원인
ssac-gen이 `@auth` 시퀀스를 `authz.Check` 호출로 변환할 때 `currentUser` (미들웨어에서 주입)의 정보를 CheckRequest에 포함하지 않음.

## 수정 제안
ssac-gen에서 `@auth` 코드 생성 시 `currentUser`의 claims를 `CheckRequest`에 포함하도록 수정.
