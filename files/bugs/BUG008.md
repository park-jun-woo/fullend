# BUG008: @call 실패 시 500 반환 — VerifyPassword 실패가 서버 에러로 처리됨

## 단계
ssac-gen

## 심각도
HIGH — 잘못된 HTTP 상태 코드

## 증상
Login에서 비밀번호 틀리면 401이 아닌 500 Internal Server Error 반환.

## 생성된 코드

```go
// internal/service/auth/login.go
if err := auth.VerifyPassword(auth.VerifyPasswordRequest{...}); err != nil {
    c.JSON(http.StatusInternalServerError, gin.H{"error": "호출 실패"})
    return
}
```

## 기대 동작
`@call auth.VerifyPassword`가 실패하면 비밀번호 불일치이므로 401 Unauthorized 반환.

## 원인
ssac-gen이 `@call` 시퀀스의 에러를 일괄적으로 500으로 처리. `auth.VerifyPassword`처럼 비즈니스 검증 성격의 @call은 4xx로 처리해야 함.

## 수정 제안
- `@call` 에러 시 기본 500이 아닌, SSaC에서 에러 코드를 지정할 수 있는 문법 추가 (예: `@call auth.VerifyPassword(...) 401 "Invalid credentials"`), 또는
- `auth.VerifyPassword` 같은 built-in verify 계열 함수는 자동으로 401로 매핑
