# BUG001: ssac-model scanXxx 변수명 충돌

## 단계
ssac-model gen

## 증상
`go build` 시 `s redeclared in this block` 컴파일 에러.

## 재현
DDL에 `schedules` 테이블이 있을 때 `fullend gen` 실행.

## 생성된 코드

```go
// internal/model/schedule.go
func scanSchedule(s interface{ Scan(...interface{}) error }) (*Schedule, error) {
    var s Schedule  // ← 파라미터 s와 재선언 충돌
    err := s.Scan(&s.ID, &s.ProjectID, ...)
```

## 기대 동작
다른 모델(document→d, message→m, project→p, user→u)처럼 충돌 없는 변수명 사용.

## 원인
codegen이 모델 타입 첫글자를 로컬 변수명으로 사용하는데, `Schedule`의 `s`가 scanner 파라미터 `s`와 겹치는 케이스를 처리하지 않음. `Session`, `Setting`, `Subscription` 등 S로 시작하는 모든 모델에서 동일 버그 발생.

## 수정 제안
- scanner 파라미터를 `row`로 변경 (모든 scanXxx 함수에서 통일), 또는
- 로컬 변수명이 파라미터와 겹칠 때 두 번째 글자까지 사용 (`sc`, `se` 등)
