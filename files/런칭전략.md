### Week 2: Show HN 런칭

#### 타이틀

```
Show HN: fullend – validate 9 SSOT sources, then generate a full-stack app from them
```

이유:
- "9 SSOT sources" → 즉시 호기심 ("9개가 뭔데?")
- "validate, then generate" → 검증 먼저, 생성은 그다음. 차별화.
- "full-stack app" → 결과물 명확

피해야 할 타이틀:
- ~~"AI-powered full-stack generator"~~ → AI 피로감
- ~~"The future of software engineering"~~ → 허풍
- ~~"SSOT orchestrator for neuro-symbolic..."~~ → 학술 과다

#### 본문 구조

```
# 문제
AI 코드 에이전트가 코드를 생성하지만, 레이어 간 불일치를 검증하지 않는다.
OpenAPI가 말하는 것과 DDL이 말하는 것이 다르면? 아무도 안 잡아준다.

# fullend가 하는 일
9개 소스를 각각 검증하고, 레이어 간 교차 검증하고, 통과하면 코드를 생성한다.

[30초 데모 GIF/영상]

# 9개 SSOT
fullend.yaml, OpenAPI, SQL DDL, SSaC, Go Model, STML, Mermaid States, OPA Rego, Hurl Tests

# chain — 기능 하나의 전 레이어 추적
[chain 출력 예시]

# 링크
GitHub: ...
Getting Started: ...
```

원칙: **AI라는 단어를 최소화**. fullend의 가치는 "AI가 만든다"가 아니라 "9개 소스의 일관성을 기계적으로 보장한다".

## 리스크와 대응

**"SSOT가 뭔지 모르겠다"**
→ Getting Started에서 개념 설명 대신 결과를 보여준다. "yaml 3개 넣으면 이 코드가 나온다."

**"학습 곡선 너무 높다"**
→ `fullend init` 스캐폴딩 기능 검토. 런칭을 막을 이유는 아님.

**"HN에서 묻힌다"**
→ 정상. 2주 후 다른 앵글(Feature Chain 글)로 2차 시도.

## 상위 전략과의 연결

```
1단 로켓: fullend 오픈소스 채택 + 논문 (명성)     ← 이 문서
2단 로켓: fullend Enterprise + 컨설팅 (매출)
3단 로켓: DABEL5 투자 유치 (비전)
```

fullend 런칭은 1단 로켓의 점화다. 명성 없이 매출 없고, 매출 없이 DABEL5 없다.
